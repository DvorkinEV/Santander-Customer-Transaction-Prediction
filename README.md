# Santander Customer Transaction Prediction: LightGBM Solution
Решение для соревнования Kaggle «Santander Customer Transaction Prediction» — предсказание вероятности совершения транзакции клиентом на основе анонимных финансовых признаков.

Результат
ROC‑AUC на валидационных данных: 0.8814–0.8900 (в зависимости от конфигурации)

Финальный сабмишен: submission_ensemble.csv

Подход: ансамблевое обучение с кросс‑валидацией и обработкой выбросов

Описание задачи
Дано:

200 анонимных числовых признаков (без названий).

Целевая переменная target:

0 — клиент не совершит транзакцию;

1 — клиент совершит транзакцию.

Дисбаланс классов: ~10 % примеров относятся к классу 1, ~90 % — к классу 0.

~200 000 обучающих примеров, ~200 000 тестовых.

Цель: максимизировать ROC‑AUC, точно предсказывая вероятность транзакции.

Ключевые этапы решения
Анализ данных

Проверка распределения целевой переменной.

Оценка статистических характеристик признаков (среднее, std, min/max).

Выявление дисбаланса классов.

Предобработка

Удаление идентификаторов (ID_code).

Обработка выбросов: усечение значений до 5‑го и 95‑го перцентиля.

Нормализация не требуется (LightGBM устойчив к масштабу).

Моделирование

LightGBM как основной алгоритм (быстрый, эффективный для табличных данных).

Настройка гиперпараметров:

scale_pos_weight=10.0 для учёта дисбаланса;

регуляризация (reg_alpha, reg_lambda);

ограничение глубины деревьев (max_depth=6).

Ранняя остановка (early_stopping) для избежания переобучения.

Валидация

Stratified K‑Fold (3 фолда) для устойчивой оценки.

Ансамбль из 3 моделей с разными random_state.

Усреднение предсказаний по фолдам и моделям.

Сабмишен

Формат: CSV с колонками ID_code и target (вероятность).

Сохранение в submission_ensemble.csv.
